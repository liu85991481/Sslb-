## 5. 总体方案

### 5.1 系统架构

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   客户端    │────▶│   网关层    │────▶│   认证服务   │
│ （用户端）  │     │  （路由/鉴权）│     │  （核心逻辑）│
└─────────────┘     └─────────────┘     └─────────────┘
                                               │
                    ┌──────────────────────────┼──────────────────────────┐
                    ▼                          ▼                          ▼
            ┌─────────────┐           ┌─────────────┐           ┌─────────────┐
            │   用户库    │           │  Token库    │           │  日志系统   │
            │（密码存储） │           │（会话管理） │           │（审计追踪） │
            └─────────────┘           └─────────────┘           └─────────────┘
```

### 5.2 核心流程

1. **注册阶段**：用户提交注册信息 → 后端密码加密（bcrypt/scrypt）→ 写入用户库
2. **登录阶段**：用户提交凭证 → 校验密码 → 生成认证Token → 返回前端
3. **访问阶段**：携带Token请求受保护资源 → 网关验证Token有效性 → 放行或拒绝
4. **登出阶段**：Token失效处理（加入黑名单/过期）→ 清除客户端状态

### 5.3 关键技术点

- **密码安全**：bcrypt加密，盐值随机，不可逆
- **Token设计**：JWT结构，包含userId/expiry/签名
- **防攻击**：登录失败次数限制、图形验证码、IP封禁策略
- **会话管理**：Token刷新机制、长短Token结合
