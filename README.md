# SSLB - 古代三省六部制多智能体协作系统

## 项目概述

SSLB（三省六部）是一个基于大语言模型的多智能体协作系统，采用中国古代"三省六部制"组织架构，模拟真实政务流转流程。系统通过文件系统实现智能体间异步通信，支持多种模型配置，具备完整的工作流调度、状态监控和自动化测试能力。

## 核心机制

### 1. 文件系统通信机制

系统采用基于文件系统的异步通信模式，每个智能体拥有标准目录结构：

```
智能体目录/
├── inbox/          # 接收任务（上级派发）
├── outbox/         # 输出结果（完成响应）
├── reports/        # 归档报告
├── archive/        # 历史任务存档
├── .status         # 当前状态文件（idle/busy/error）
├── .env            # 环境变量（API配置）
└── CLAUDE.md       # 角色定义与提示词
```

**通信流程**：
1. 上级智能体在 `inbox/` 创建 JSON 任务文件
2. 启动脚本检测 inbox 变化，生成 prompt 并调用 API
3. 处理结果写入 `outbox/`
4. 任务文件移入 `archive/`
5. `.status` 更新状态

### 2. 消息总线路由

`.bus/routes.json` 定义智能体间的消息流转路径：

```json
{
  "routes": {
    "zhongshusheng": ["menxiasheng", "shangshusheng"],
    "menxiasheng": [],
    "shangshusheng": ["libu", "hubu", "liyibu", "bingbu", "xingbu", "gongbu"],
    "libu": [], "hubu": [], "liyibu": [],
    "bingu": [], "xingbu": [], "gongbu": []
  }
}
```

**路由规则**：
- 中书省方案 → 门下省审核、尚书省分发
- 门下省为终审节点，不再转发
- 尚书省拆解任务 → 分发给六部执行
- 六部为执行终端，不再向下传递

### 3. 消息总线调度器

`dispatcher.py` 实现自动化调度：
- 持续轮询各智能体 `outbox/`（每秒）
- 根据路由规则将结果转发至上级
- 启动对应 `.bat` 脚本唤醒智能体
- 支持两种模式：持续监听 / 主动派发
- 归档消息到 `.bus/archive/`

### 4. 结果提取机制

`extract_result.py` 从模型原始输出中提取结构化数据：
- 识别并提取 JSON 块
- 生成标准化 result 文件
- 可选生成 Markdown 报告
- 构建 artifacts 产物清单

## 工作流程

### 标准任务流转

```
1. 人类管理员 → 中书省（inbox）
       ↓
2. 中书省制定总体方案
       ↓
3. 中书省 → 门下省（审核）
       ↓
4. 门下省审核通过/驳回
       ↓
5. 中书省 → 尚书省（分发）
       ↓
6. 尚书省拆解任务 → 六部（inbox）
       ↓
7. 六部各自执行 → outbox
       ↓
8. 尚书省整合结果 → 最终交付
```

### 启动方式

1. **智能体启动**：`start-claude.bat` 检测 inbox 任务，自动处理
2. **调度器启动**：`dispatcher.py` 持续轮询消息总线
3. **中枢启动**：`core/start.bat` 启动独立维护会话

## 技术特点

### 异步解耦设计

- **文件系统通信**：不依赖网络服务，本地即可运行
- **标准化接口**：JSON 格式，兼容任意模型后端
- **独立状态**：每个智能体独立运行，互不阻塞
- **容错机制**：任务归档，失败可追溯

### 可扩展性

- **新增智能体**：创建标准目录结构 + CLAUDE.md + start-claude.bat
- **路由扩展**：修改 `.bus/routes.json` 即可
- **模型替换**：修改 `.env` 或 `settings.local.json` 无需改代码

### 技术栈

| 组件 | 技术 |
|------|------|
| 通信机制 | 文件系统（JSON） |
| API 调用 | Claude Code / requests |
| 调度系统 | Python + 轮询机制 |
| 状态管理 | JSON 文件 |
| 测试框架 | Python |
| 监控工具 | Python |
